Model       → your data (BattleState, pokemon, DB)
    Type
        id
        name

    StatusEffect
        id
        name
        description
        is_volatile

    Move
        id
        name
        type -> Type
        category (physical/special/status)
        power
        accuracy
        pp
        description
        statis_effect -> StatusEffect
        status_chance

    PokemonSpecies  (fixed, from DB)
        id (pokedex number)
        name
        type1 -> Type
        type2 -> Type?
        base_hp, base_atk, base_def
        base_spatk, base_spdef, base_speed
        learnable_moves[] -> Move

    PokemonInstance (live pokemon in a team)
        species -> PokemonSpecies
        computed_hp, computed_atk, computed_def,
        computed_spatk, computed_spdef, computed_speed
        moves[4] -> Move
        current_pp[4]

    BattlePokemon   (PokemonInstance during a battle)
        instance -> PokemonInstance
        current_hp
        status -> StatusEffect
        stat_stages[6]  (atk/def/spatk/spdef/speed/acc, -6 to +6)

    BattleState
        player[6] -> BattlePokemon
        enemy[6] -> BattlePokemon
        player_current
        enemy_current
        opponent -> Opponent
        weather -> StatusEffect
        weather_turns_left (-1 = permanent)
        phase (SELECT_MOVE, ANIMATING, SWITCH_IN, GAME_OVER)

    Opponent    (Interface)
        AIOpponent
        LocalOpponent
        NetworkOpponent

    GameContext (shared across screens)
        db -> Sqlite
        battle -> BattleState
        opponent -> Opponent
        player_team[6] -> PokemonInstance
